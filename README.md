# Doc Summarize with LM Studio

Плагин для Obsidian, который использует LM Studio API для суммаризации документов, изображений и заметок, а также для ответов на вопросы о них. Поддерживает распознавание патентов, работу с индексами файлов и сохранение чатов.

## Возможности

- **Суммаризация документов** - быстрое создание резюме текущего документа или выделенного текста
- **Чат о документе** - задавайте вопросы о содержимом заметки с поддержкой прикрепленных файлов
- **Анализ изображений** - описание и анализ изображений (требуется модель с vision, например LLaVA)
- **Поиск по vault** - задавайте вопросы по всем заметкам с автоматическим поиском релевантных
- **Связанные заметки** - автоматическое включение связанных заметок в контекст
- **Ссылки на заметки** - кликабельные ссылки [[заметка]] в ответах
- **Распознавание патентов** - автоматическое преобразование патентных номеров в кликабельные ссылки (US, RU, CN, JP)
- **Индекс файлов** - создание полного индекса всех документов vault с превью содержимого
- **Предотвращение дубликатов** - автоматическое исключение повторяющихся файлов
- **Сохранение чатов** - экспорт разговоров с прикрепленными файлами как заметки Obsidian
- **Поддержка всех типов файлов** - работа с текстом, изображениями, видео и другими форматами

## Требования

1. [LM Studio](https://lmstudio.ai/) - запущенный локально
2. Загруженная модель в LM Studio (например, Llama, Mistral, и т.д.)
3. Включённый сервер в LM Studio (Local Server)

## 🔧 Установка

### Ручная установка

1. Скачайте файлы плагина из [репозитория на GitHub](https://github.com/Processori7/doc-summarize-lmstudio.git)
2. Создайте папку `doc-summarize-lmstudio` в `<vault>/.obsidian/plugins/`
3. Скопируйте `main.js`, `manifest.json` и `styles.css` в эту папку
4. Перезапустите Obsidian
5. Включите плагин в настройках (Settings → Community plugins)

### Сборка из исходников

```bash
# Установка зависимостей (установить необходимые пакеты)
npm install

# Сборка для разработки (отслеживает изменения файлов)
npm run dev

# Сборка для продакшена (финальная сборка)
npm run build
```

## Настройка

1. Откройте настройки Obsidian → Doc Summarize
2. Укажите URL LM Studio API (по умолчанию: `http://localhost:1234/v1`)
3. Настройте промпты и параметры генерации по желанию

### Настройки индекса файлов

- **Использовать индекс**: Включает создание индекса при добавлении файлов
- **Максимальное количество файлов**: Ограничение на количество файлов в индексе
- **Фильтры**: По папкам, тегам, возрасту файлов
- **Максимальный контекст**: Ограничение на размер контекста для AI

### Настройки патентов

- Автоматическое распознавание включено по умолчанию
- Поддержка форматов: US, RU, CN, JP
- Ссылки ведут на Google Patents

## Использование

### Команды (Палитра команд - Ctrl/Cmd + P)

| Команда | Описание |
|---------|----------|
| `Суммаризировать текущий документ` | Создаёт резюме всего документа |
| `Суммаризировать выделенный текст` | Создаёт резюме выделенного фрагмента |
| `Задать вопрос о документе` | Открывает чат для вопросов о текущей заметке |
| `Описать/суммаризировать изображение` | Анализ изображения (требуется vision-модель) |
| `Задать вопрос о заметках` | Поиск и вопросы по всему vault |
| `Вставить суммаризацию в документ` | Генерирует и вставляет резюме прямо в документ |
| `Добавить все файлы из папки` | Добавляет файлы из выбранной папки с индексом |
| `Сохранить чат как заметку` | Экспортирует текущий чат в новую заметку |

### Горячие клавиши

Горячие клавиши можно назначить для любой команды в настройках Obsidian (Настройки → Горячие клавиши).

## Работа с изображениями

Для анализа изображений требуется модель с поддержкой vision:
- LLaVA (рекомендуется)
- BakLLaVA
- Другие мультимодальные модели

## Связанные заметки

Плагин автоматически включает содержимое связанных заметок в контекст:
- Настраивается в параметрах
- Глубина поиска: 1-3 уровня
- Помогает получать более полные ответы

## Распознавание патентов

Плагин автоматически распознает патентные номера в тексте и преобразует их в кликабельные ссылки на Google Patents:
- **Поддерживаемые форматы**: US (США), RU (Россия), CN (Китай), JP (Япония)
- **Пример**: `**US1234567A**` → [US1234567A](https://patents.google.com/patent/US1234567A)
- Работает в ответах AI, системных сообщениях и сохраненных чатах

## Индекс файлов

Создание полного индекса всех документов в vault:
- Включает все типы файлов (текст, изображения, видео и др.)
- Показывает превью содержимого для текстовых файлов
- Отображает размер и тип для бинарных файлов
- Автоматическая фильтрация по папкам, тегам и дате изменения
- Помогает ориентироваться в больших коллекциях документов

## Сохранение чатов

Экспорт разговоров в формате заметок Obsidian:
- Сохраняет всю историю сообщений
- Включает список прикрепленных файлов
 - Поддерживает форматирование markdown
- Создает отдельную заметку для каждого чата

Плагин полностью на русском языке. Промпты можно настроить в параметрах.

## Устранение неполадок

### LM Studio не подключается

1. Убедитесь, что LM Studio запущен
2. Проверьте, что Local Server включён (вкладка Local Server → Start Server)
3. Проверьте URL в настройках плагина
4. Используйте кнопку "Проверить подключение" в настройках

### Медленная генерация

- Уменьшите максимальное количество токенов
- Используйте более лёгкую модель
- Отключите включение связанных заметок

### Ошибки при анализе изображений

- Убедитесь, что используется модель с поддержкой vision (LLaVA и т.д.)
- Проверьте размер изображения (слишком большие могут вызывать проблемы)

### Проблемы с индексом файлов

- Проверьте настройки фильтров (папки, теги)
- Увеличьте максимальный контекст если индекс обрезается
- Для больших vault используйте ограничение на количество файлов

### Патентные ссылки не работают

- Убедитесь, что патентный номер в формате **XX1234567A**
- Проверьте подключение к интернету для ссылок на Google Patents
- Поддерживаемые страны: US, RU, CN, JP
